Responsywność

Responsywność to zdolność strony internetowej do adaptacji do różnych rozdzielczości ekranu i urządzeń (telefonów, tabletów, komputerów, itp.).

Aby strona była responsywna, musi zawierać:

  1. <meta name="viewport" content="width=device-width, initial-scale=1.0"> w sekcji <head> - instrukcje dotyczące sposobu kontrolowania wymiarów i skalowania strony
  2. media queries (@media) na co najmniej 3 urządzenia

@media

Media queries to reguły CSS, które pozwalają na stosowanie różnych stylów w zależności od warunków urządzenia lub okna przeglądarki.

W media queries pisze się "drugiego CSS" używając reguł stylów (selektor i deklaracje). Można stosować je włącznie do ekranów urządzeń (screen), drukarek (print) lub wszystkich (all - domyślnie).

Najczęściej wykorzystywane warunki:

  • max-width - maksymalna szerokość okna przeglądarki
  • min-width - minimalna szerokość okna przeglądarki
  • width - szerokość okna przeglądarki
  • max-height - maksymalna wysokość okna przeglądarki
  • min-height - minimalna wysokość okna przeglądarki
  • height - wysokość okna przeglądarki
  • orientation - orientacja urządzenia (landscape lub portrait)
  • prefers-color-scheme - preferencje dotyczące schematu kolorów (light lub dark)

Przykład:

@media screen and (max-width: 768px) {
    body {
        font-size: 14px;
    }
}

<picture>

Element <picture> pozwala na wyświetlanie różnych wersji obrazu w zależności od warunków urządzenia lub okna przeglądarki, aby zmniejszyć obciążenie strony. Zamiast jednego skalowanego obrazu można zastosować wiele obrazów, które wyświetlą się w odpowiednich dla nich warunkach.

Przykład:

<picture>
    <source media="(max-width: 768px)" srcset="image-mobile.jpg">
    <source media="(max-width: 1024px)" srcset="image-tablet.jpg">
    <img src="image-desktop.jpg" alt="Opis obrazu">
</picture>
  1. <source> określa różne wersje obrazu dla różnych warunków
  2. srcset - obowiązkowy atrybut definiujący źródło obrazu
  3. media - atrybut definiujący warunki wyświetlania obrazu

Należy również zdefiniować <img> dla przeglądarek, które nie obsługują <picture>.
